# 什么是CSS3

CSS3是CSS（层叠样式表）技术的升级版本，于1999年开始制订，2001年5月23日W3C完成了CSS3的工作草案，主要包括盒子模型、列表模块、超链接方式、语言模块、背景和边框、文字特效、多栏布局等模块

# 插入样式的三种方法

  1. 外部样式表：通过link元素引入外部样式表
<link rel="stylesheet" type="text/css" href="">
  1. 内部样式表（style中）
<style type="text/css"></style>
  1. 内联样式表（行内）
<p type="color:red"></p>

# 语法

  1. 单一元素的多个属性之间用分号隔开；若只有一个可以省略

  2. 多个元素之间用逗号隔开

  3. 如果值为若干单词，则要给值加引号;除了这种情况外其他时候不可以加引号

  4. 样式的优先顺序:

  • 设计者设计的样式>浏览器用户自定义的样式>浏览器自设的样式
  • 行内（内联）样式>内部样式：style中的>外部样式
  • 强制优先级：!important
  1. 层叠、继承、冲突

外观样式--比如字体、颜色可以继承;而布局有关的样式不可以继承，比如边框等

层叠性

# 优先级的判断

  • 权重的计算公式：

行内样式：1000
id选择器：0100
类选择器：0010 伪类选择器：0010 属性选择器：0010 伪元素选择器：0010
标签选择器：0001

  • 结构就近原则：最后加载的会覆盖前面加载的样式；

  • 书写就近原则：{color:red;color:blue;}后写的会覆盖先写的属性；

# 样式私有化

在单页面应用中每一个打开过的样式都会加载到全局样式里。 通过<style>标签的scoped属性可以规定样式的私有作用域比较样式污染。

<style type="text/css" scoped>
</style>
  • Vue下样式私有化原理
  1. 在Vue项目中使用scoped时，Vue会在渲染DOM时为每一个设置了私有化样式的DOM增加一个data-v-id的自定义属性作为全局标识。
<div class="list">
    <div data-v-3b648845="" class="tagtree"></div>
</div>
  1. 在设置了私有化样式scoped时，Vue会自动给最后一个选择器后面追加一个属性选择器，实现与绑定了data-v-id的DOM一一对应
/* vue自动给`.tagtree`追加属性选择器而不是`.list` */
.list .tagtree[data-v-3b648845] {
    color:red;
}

# 样式穿透

当我们使用第三方组件库且使用私有化样式scoped时，此时在修改第三方组件样式时会出现以下问题:

Vue在构建时(有时只会给<el-input>增加data-v-id标识)不会给内部的<input>标签增加data-v-id

<div class="list" data-v-3b648845="">
    <el-input data-v-3b648845="" class="myTxt">
        <input type="text">
    </el-input>
</div>

但是此时会在<input>选择器后面增加一个data-v-id属性选择器，此时样式就不会生效。

.list input[data-v-3b648845] {
    color:red;
}

# 深度选择器

本质就是修改data-v-id属性选择器在CSS选择其中的位置

  • >>>写法：
.list >>> input {
    color:red;
}
  • ::v-deep写法：
.list ::v-deep input {
    color:red;
}
  • /deep/写法：
.list /deep/ input {
    color:red;
}
  • :deep()写法：
:deep(.list)  input {
    color:red;
}

渲染后的样式，此时他会放在使用了深度选择器的位置

.list[data-v-3b648845] input {
    color:red;
}

各深度选择器的区别

名称 适用场景
>>> 原生CSS
::v-deep 预处理器（vue-cli3以上不支持）
/deep/ 预处理器（Vue3不支持）
:deep() 预处理器（支持Vue3）
