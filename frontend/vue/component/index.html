# 6.4.1 组件注册

# 组件名

  • kebab-case
    使用kebab-case(短横线分隔命名)定义组件时，你也必须在引用这个自定义元素时使用kebab-case
    例如:<my-component-name>
  • PascalCase 当使用PascalCase(首字母大写命名)定义一个组件时,引用是kebab-case/PascalCase都可以。

# 组件的作用域

  • 全局注册
    可以用在任何新创建的Vue根实例
  • 局部注册
    只能用在当前组件内，且不能用在其子组件中。

# 模块系统

将每个组件放置在其各自的文件中。
通过import/require引入组件。（ES6 Module详见7.16.20 CommonJS详见7.1.6）
全局注册的行为必须在根Vue实例(通过 new Vue)创建之前发生. 局部注册：

import BaseButton from './BaseButton.vue'
import BaseIcon from './BaseIcon.vue'
import BaseInput from './BaseInput.vue'

export default {
  components: {
    BaseButton,
    BaseIcon,
    BaseInput
  }
}

# 6.4.2 全局组件

  1. 第一步声明
new Vue({el:`#app`})
  1. 第二步全局注册挂载
Vue.component("Vheader",{template:`<div>我是模板</div>`});
 //Vue.component(组件名称 template:`模板（需要一个根元素，否则只能渲染第一个元素）`)
  1. 第三步使用
<div id="app">
<Vheader></Vheader><!--双标签 -->
<Vheader/><!--单标签（用于全局组件，el标记的组件只会渲染第一个） -->
</div>

# 6.4.3 局部组件

  1. 第一步声明
new Vue({el:"#app"});
  1. 第二步局部注册挂载，仅用于#app下局部使用
new Vue({
    el:"#app",
    components: {
        Vheader:{template:`<div>子组件</div>`}//3.组件描述
    },
});
  1. 第三步使用组件
<div id="app">
    <Vheader></Vheader>
</div>

WARNING

  1. 如果子组件在外部描述需要先声明，然后再父组件中注册挂载
  2. 子组件的data必须是一个函数,否则在复用组件时，将公用一个data

# keep-alive组件缓存

参数：

include：（捕获需要缓存的）字符串、数组或正则表达式。只有名称匹配的组件会被缓存。
exclude：（捕获不需要缓存的）字符串、数组或正则表达式。任何名称匹配的组件都不会被缓存。
max：数字。最多可以缓存多少组件实例。

// cachedViews 需要缓存的路由name数组
<keep-alive :include="cachedViews">
  <component v-bind:is="currentTabComponent"></component>
</keep-alive>
// 缓存路由：keep-alive与router-view结合
<keep-alive :include="cachedViews">
  <router-view :key="key"></router-view>
</keep-alive>

WARNING

  1. keep-alive要求被切换到的组件都有自己name
  2. keep-alive与router-view结合使用时，include优先匹配组件的name所以组件的name必须与路由的name完全一致
  3. exclude与includet同时使用时，exclude的优先级会更高

缓存组件的生命周期：

activated被keep-alive缓存的组件激活时调用 deactivated被keep-alive缓存的组件停用时调用

TIP

运行顺序：created→activated→deactivated→activated

# 父传子props

  • 父组件
<template>
  <div>
    <input type="text" v-model="name">
    <!-- 引入子组件,通过自定义属性inputName传递数据name -->
    <child :inputName="name"></child>
  </div>
</template>
<script>
  import child from './child'
  export default {
    components: {
      child
    },
    data () {
      return {
        name: 'jaqi'
      }
    }
  }
</script>
  • 子组件
<template>
  <div>
    <span>{{inputName}}</span>  <!-- jaqi -->
  </div>
</template>
<script>
  export default { 
    props: {
      inputName: String, // 通过自定义的inputName属性接受父组件传来的字符串(String)类型的name
    }
  }
</script>

props

props是单向下行绑定，父组件更新回传给子组件，反之不行
Prop可以验证，并设置默认值。如果验证失败开发环境回产生一个控制台警告

 props: {
    // Number类型验证
    propA: Number,
    // 多个类型验证
    propB: [String, Number],
    // 必填的String类型
    propC: {
      type: String,
      required: true
    },
    // 带有默认值的Number类型
    propD: {
      type: Number,
      default: 100
    },
    // 带有默认值的Object
    propE: {
      type: Object,
      // 对象或数组默认值必须从一个工厂函数获取
      default: function () {
        return { message: 'hello' }
      }
    },
    // 自定义验证函数
    propF: {
      validator: function (value) {
        // 这个值必须匹配下列字符串中的一个
        return ['success', 'warning', 'danger'].indexOf(value) !== -1
      }
    }
  }

# 子传父emit

  • 子组件
<template>
  <div>
    <span>{{childValue}}</span>
    <!-- 定义一个子组件传值的方法 -->
    <input type="button" value="点击触发" @click="childClick">
  </div>
</template>
<script>
  export default {
    data () {
      return {
        childValue: '我是子组件的数据'
      }
    },
    methods: {
      childClick () {
        // childByValue是在父组件on监听的方法
        // 第二个参数this.childValue是需要传的值
        this.$emit('childByValue', this.childValue)
      }
    }
  }
</script>
  • 父组件
<template>
  <div>
    <span>{{name}}</span>
    <br>
    <br>
    <!-- 引入子组件 定义一个on的方法监听子组件的状态-->
    <child v-on:childByValue="childByValue"></child>
  </div>
</template>
<script>
  import child from './child'
  export default {
    components: {
      child
    },
    data () {
      return {
        name: ''
      }
    },
    methods: {
      childByValue: function (childValue) {
        // childValue就是子组件传过来的值
        this.name = childValue
      }
    }
  }
</script>

# 兄弟传值$emit与$on

  • 组件A
<template>
  <div>
    <span>{{elementValue}}</span>
    <input type="button" value="点击触发" @click="elementByValue">
  </div>
</template>
<script>
  // 引入公共的bug，来做为中间传达的工具
  import Bus from './bus.js'
  export default {
    data () {
      return {
        elementValue: 4
      }
    },
    methods: {
      elementByValue: function () {
        Bus.$emit('val', this.elementValue) // 通过$emit存入Bus
      }
    }
  }
</script>
  • 组件B
<template>
  <div>
    B组件:
    <input type="button" value="点击触发" @click="getData">
    <span>{{name}}</span>
  </div>
</template>
<script>
  import Bus from './bus.js'
  export default {
    data () {
      return {
        name: 0
      }
    },
    mounted: function () {
      var vm = this
      // 通过$on从Bus取
      Bus.$on('val', (data) => {
        console.log(data)
        vm.name = data
      })
    },
    methods: {
      getData: function () {
        this.name++
      }
    }
  }
</script>

# 父传后代provide与inject（依赖注入）

  • 父组件
<template>
  <div>
    <input type="text" v-model="name">
    <br>
    <my-child></my-child>
  </div>
</template>
<script>
  import mychild from './child'
  export default {
    components: {
      mychild
    },
    // 父组件通过provide方法想数据分发给后代组件
     provide() { 
        return {
            name: 'jaqi'
        };
    },
  }
</script>
  • 后代组件
<template>
 <div class="views__home__my-children">
        <my-grandson></my-grandson>
    </div>
</template>
<script>
  import mygrandson from './grandson'
  export default {
    components: {
      mygrandson
    },
    // 后代组件直接通过inject获取父组件分发的数据
    inject: ['name'],
  }
</script>

# 6.4.6 组件其他方法

# 在组件上使用v-model

<ChildComponent v-model="searchText"/>

子组件props必须绑定名为value的变量。并在input事件中通过$emit返回给父组件

Vue.component('ChildComponent ', {
  props: ['value'],
  template: `
    <input
      v-bind:value="value"
      v-on:input="$emit('input', $event.target.value)"
    >
  `
})
  • 使用model选项 允许一个自定义组件在使用v-model时定制prop和event命名
Vue.component('ChildComponent ', {
  model: {
    prop: 'searchText',
    event: 'change'
  },
  props: {
    // 这将允许 `value` 属性用于其他用途
    value: String,
    // 使用searchText代替value作为model的prop
    searchText: {
      type: Number,
      default: 0
    }
  },
  // ...
})
  • *Vue 3.x中语法

# 有参数的v-model

<!-- 使用searchText代替value作为model的prop -->
<!-- 使用update:searchText作为$event的事件名 -->
<ChildComponent v-model:searchText="title" />

# 无参数的v-model

prop和event命名更改为modelValue和update:modelValue

<ChildComponent v-model="title" />

// ChildComponent.vue
export default {
  props: {
    modelValue: String // Vue 2.x是`value：String`
  },
  emits: ['update:modelValue'],
  methods: {
    changePageTitle(title) {
      this.$emit('update:modelValue', title) // Vue 2.x是 `this.$emit('input', title)`
    }
  }
}

# 混入(mixin)

将任意组件通过mixin包装成混入对象，插入另一个组件,

// 定义一个混入对象
var mixin = {
  created: function () {
    console.log('混入对象的钩子被调用')
  }
}
// 使用混入对象
new Vue({
  mixins: [mixin],
  created: function () {
    console.log('组件钩子被调用')
  }
})
  • 使用Vue.extend进行混入
// 定义一个混入对象
var myMixin = {
  created: function () {
    this.hello()
  },
  methods: {
    hello: function () {
      console.log('hello from mixin!')
    }
  }
}

// 定义一个使用混入对象的组件
var Component = Vue.extend({
  mixins: [myMixin]
})

var component = new Component() // => "hello from mixin!"

TIP

  1. 当组件和混入对象含有同名选项时,methods、components和directives，将被合并为同一个对象。两个对象键名冲突时，取组件对象的键值对。
  2. *Vue 3.x中合并操作将被浅层次地执行，详见mixin-合并行为变更 (opens new window)
  • 全局混入
Vue.mixin({
  created: function () {
    console.log('全局混入')
  }
})
  • 自定义选项合并策略 Vue.config.optionMergeStrategies自定义合并策略
const merge = Vue.config.optionMergeStrategies.computed
Vue.config.optionMergeStrategies.vuex = function (toVal, fromVal) {
  if (!toVal) return fromVal
  if (!fromVal) return toVal
  return {
    getters: merge(toVal.getters, fromVal.getters),
    state: merge(toVal.state, fromVal.state),
    actions: merge(toVal.actions, fromVal.actions)
  }
}

# 控制更新

由于JavaScript的限制，Vue不能检测数组和对象的变化。 因此在变更数组和对象的时候可以使用以下两种方法触发视图更新：

  1. $forceUpdate强制视图更新。（只会影响实例本身和插入插槽内容的子组件，而不是所有子组件）
this.$forceUpdate();
  1. 通过$set方法修改数组或对象触发视图更新。
this.$set(target,key,value); // target = Object | Array
